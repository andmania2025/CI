# コードレビュープロンプト

## レビューの重点分野

### 1. コード品質
- コードは確立されたルールに従っているか？
- コードは読みやすく、構造化されているか？
- エラーハンドリングは適切か？

### 2. セキュリティ
- セキュリティ上の脆弱性はないか？
- ユーザー入力は適切に検証されているか？
- 認証と認可は正しく実装されているか？

### 3. パフォーマンス
- パフォーマンスのボトルネックはないか？
- React hooksと状態管理を効率的に使用しているか？
- データベースクエリは最適化されているか？

### 4. テスト
- コードはテスト可能か？
- 十分なテストがあるか？

### 5. アーキテクチャ
- コードはアーキテクチャパターンに従っているか？
- 関心の分離は維持されているか？

## レビューチェックリスト

- [ ] TypeScriptのベストプラクティスに従っている
- [ ] 適切な命名規則を使用
- [ ] Server Actionsに適切なエラーハンドリング
- [ ] Prismaを適切に使用
- [ ] 認証が適切に検証されている
- [ ] Tailwindクラスを一貫して使用
- [ ] テストが存在し意味がある
- [ ] Propsインターフェースが明確
